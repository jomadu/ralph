# Task: Build Ralph - A Simple AI Loop Runner

## What is Ralph?

Ralph is the original [Ralph Loop](https://ghuntley.com/ralph/) concept, productized. A dumb loop that pipes a prompt file to an AI CLI, lets it work, lets it exit, and repeats — with proper loop control, error handling, signal scanning, and configurable prompt aliases.

Ralph strips away rooda's opinionated structure (OODA phases, fragment composition, 16 typed procedures, AGENTS.md lifecycle) and keeps the core insight: fresh AI process per iteration + file-based state continuity.

**Ralph is a loop runner, not a methodology.**

## Core Design Principles

- **Unopinionated about prompt content.** Ralph doesn't know or care what's inside the prompt file. The user owns the prompt entirely.
- **One alias = one file = one prompt.** No fragment arrays, no phase decomposition, no composition engine.
- **Preamble injection.** Ralph wraps the user's prompt with loop metadata (iteration, signals, context) before piping it.
- **Configurable signals.** The success/failure signal strings are configurable per prompt, not hardcoded.
- **Same loop mechanics as rooda.** Fresh AI process per iteration, `<promise>` signal scanning, consecutive failure tracking, timeouts, max iterations, all the proven loop infrastructure.

## What Ralph Includes

### 1. Iteration Loop
- Fresh AI CLI process per iteration (clear context between runs)
- Signal scanning on captured output (configurable signal strings)
- Consecutive failure tracking with configurable threshold
- Max iterations with configurable limit
- Unlimited mode (runs until signal, failure threshold, or Ctrl+C)
- Iteration timeout (kill process if it exceeds duration)
- Output buffer with configurable max size (truncate from beginning, keep recent output for signal scanning)
- SIGINT/SIGTERM handling (kill AI CLI, wait with timeout, exit 130)
- Iteration statistics at completion (count, min, max, mean, stddev via Welford's algorithm)
- Dry-run mode (validate config, display assembled prompt, don't execute)
- Exit codes: 0 (success), 1 (aborted/failure threshold), 2 (max-iters reached), 130 (interrupted)

### 2. AI CLI Integration
- Pipe assembled prompt (preamble + prompt file) to AI CLI stdin
- Configurable AI command via direct command string or named alias
- Built-in aliases: `kiro-cli`, `claude`, `copilot`, `cursor-agent`
- User-defined aliases in config (merge with built-in)
- Capture stdout/stderr to buffer
- Stream output to terminal in verbose mode (while still capturing for signal scanning)
- Shell-style command parsing (quoted args supported, no shell execution)
- Process timeout and crash recovery (partial output captured)
- AI CLI process inherits environment variables and working directory

### 3. Configuration (Three-Tier)
- **Built-in defaults** — compiled into binary, always available
- **Global config** — `~/.config/ralph/ralph-config.yml` (resolved via `RALPH_CONFIG_HOME` > `$XDG_CONFIG_HOME/ralph/` > `~/.config/ralph/`)
- **Workspace config** — `./ralph-config.yml`
- **Environment variables** — `RALPH_*` prefix
- **CLI flags** — highest precedence
- Precedence: CLI flags > env vars > workspace > global > built-in defaults
- Provenance tracking (know where each resolved value came from)
- Missing config files silently skipped
- Unknown keys produce warnings (forward compatibility)
- Config validated at load time (fail fast)

### 4. Prompt Aliases
- Config maps a name to a prompt file path
- Each prompt can override loop configuration
- `ralph run <alias>` reads the file and feeds it to the loop
- No built-in prompts — bring your own

### 5. Preamble Injection
- Generated by the loop, wraps the user's prompt file content
- Includes: iteration number, max iterations (or "unlimited"), prompt name
- Includes: signal instructions (generated from the prompt's signal config, so the AI knows what to emit)
- Includes: user-provided context (via `--context` flag)
- Enabled by default (`loop.preamble: true`)
- Can be disabled globally (`loop.preamble: false`) or per-prompt (`prompts.<name>.loop.preamble: false`)

#### Preamble Format

Limited iterations:
```
───────────────────────────────────────────
RALPH LOOP EXECUTION
───────────────────────────────────────────

Iteration: 3/10
Prompt: build

Signals:
- When your task is complete, output: <promise>SUCCESS</promise>
- If you are blocked, output: <promise>FAILURE</promise>

───────────────────────────────────────────
CONTEXT
───────────────────────────────────────────
Focus on the auth module, JWT validation is broken

───────────────────────────────────────────
PROMPT
───────────────────────────────────────────
[contents of prompt file]
```

Unlimited iterations:
```
Iteration: 3 (unlimited)
```

No context provided: CONTEXT section omitted entirely.

Preamble disabled: prompt file content piped directly, no wrapping.

### 6. Configurable Signals
- Default signals: `<promise>SUCCESS</promise>` and `<promise>FAILURE</promise>`
- Configurable at `loop.signals.success` and `loop.signals.failure`
- Overridable per-prompt at `prompts.<name>.loop.signals.success` / `.failure`
- Preamble signal instructions generated from the configured signal strings
- Loop scanner searches for the configured strings (substring match)
- Signal precedence: failure wins if both present in output
- Signal scanning happens after AI CLI exits (not during streaming)

## What Ralph Does NOT Include

- No OODA phase structure (no observe/orient/decide/act decomposition)
- No fragment composition system (no arrays of fragments concatenated per phase)
- No built-in procedure library (no hardcoded workflows)
- No embedded prompt files (bring your own)
- No AGENTS.md format, lifecycle, or bootstrap
- No documentation lifecycle or humanizer skill
- No work tracking integration
- No planning system

## Configuration Schema

### Config File: `ralph-config.yml`

```yaml
loop:
  iteration_mode: max-iterations       # "max-iterations" or "unlimited" (default: max-iterations)
  default_max_iterations: 5            # Default iteration limit (default: 5, must be >= 1)
  failure_threshold: 3                 # Consecutive failures before abort (default: 3)
  iteration_timeout: 300               # Per-iteration timeout in seconds (nil = no timeout)
  max_output_buffer: 10485760          # Max output buffer in bytes (default: 10MB)
  log_level: info                      # debug, info, warn, error (default: info)
  show_ai_output: false                # Stream AI CLI output to terminal (default: false)
  ai_cmd_alias: claude                 # Default AI command alias
  # ai_cmd: "custom-cli --flags"       # Or direct command (overrides ai_cmd_alias)
  preamble: true                       # Inject preamble into prompt (default: true)
  signals:
    success: "<promise>SUCCESS</promise>"   # Success signal string (default)
    failure: "<promise>FAILURE</promise>"   # Failure signal string (default)

ai_cmd_aliases:
  claude: "claude -p --dangerously-skip-permissions"
  kiro: "kiro-cli chat --no-interactive --trust-all-tools"
  copilot: "copilot --yolo"
  cursor-agent: "cursor-agent -p -f --stream-partial-output --output-format stream-json"

prompts:
  build:
    path: "./prompts/build.md"
    name: "Build"
    description: "Run the main build loop"
    loop:
      default_max_iterations: 10
      failure_threshold: 5

  bootstrap:
    path: "./prompts/bootstrap.md"
    name: "Bootstrap"
    description: "One-shot project setup"
    loop:
      default_max_iterations: 1
      preamble: false

  custom-agent:
    path: "./prompts/custom.md"
    name: "Custom Agent"
    description: "Run a custom task with Kiro"
    loop:
      default_max_iterations: 3
      ai_cmd_alias: kiro
      preamble: false
      signals:
        success: "TASK_COMPLETE"
        failure: "TASK_BLOCKED"
```

### Prompt Config Structure

Each prompt has three fields of its own: `path`, `name`, and `description`. Everything else is `prompts.<name>.loop.*`, which directly overrides `loop.*`.

```yaml
prompts:
  <name>:
    path: string              # Path to prompt file (required)
    name: string              # Display name for listing (optional, defaults to alias key)
    description: string       # Short description for listing (optional)
    loop:                     # Loop overrides (all optional, inherit from root loop)
      iteration_mode: string
      default_max_iterations: int
      failure_threshold: int
      iteration_timeout: int
      max_output_buffer: int
      ai_cmd: string
      ai_cmd_alias: string
      preamble: bool
      signals:
        success: string
        failure: string
```

### CLI Flags

```
ralph run <prompt-alias> [flags]

Loop Control:
  --max-iterations, -n <int>    Override max iterations
  --unlimited, -u               Run until signal or failure threshold
  --failure-threshold <int>     Consecutive failures before abort
  --iteration-timeout <int>     Per-iteration timeout in seconds (0 = no timeout)
  --max-output-buffer <int>     Max output buffer in bytes
  --preamble                    Inject preamble into prompt (default: true)
  --no-preamble                 Disable preamble injection
  --dry-run, -d                 Validate and display assembled prompt, don't execute

AI Command:
  --ai-cmd <string>             Direct AI command string
  --ai-cmd-alias <string>       AI command alias name

Signals:
  --signal-success <string>     Success signal string
  --signal-failure <string>     Failure signal string

Context:
  --context, -c <string>        Inject context into preamble (repeatable, file or inline)

Output Control:
  --verbose, -v                 Stream AI output + debug logging
  --quiet, -q                   Suppress non-error output
  --log-level <string>          Set log level (debug, info, warn, error)

Configuration:
  --config <path>               Alternate workspace config file path

Other:
  ralph list                    List available prompt aliases
  ralph version                 Show version info
  --help, -h                    Show help
```

## Comparison with Rooda

| Aspect | Rooda | Ralph |
|---|---|---|
| Unit of work | Procedure = 4 OODA phases x N fragments | Prompt alias = 1 file |
| Prompt assembly | Compose preamble + context + 4 phase sections from fragment arrays | Read a file, wrap with preamble, pipe it |
| Built-in content | 58 embedded fragments, 16 procedures | No embedded prompts. Bring your own. |
| Opinionation | Highly opinionated (OODA, AGENTS.md, JTBD specs) | Unopinionated. Loop runner, not a methodology. |
| Signals | Hardcoded `<promise>SUCCESS/FAILURE</promise>` | Configurable per-prompt |
| Preamble | Always on, OODA-framed with phase descriptions | On by default, opt-out per prompt, generic framing |
| Config complexity | Procedures with fragment arrays, template parameters | Prompts map names to files, optional loop overrides |

- Dependencies: cobra (CLI), go-shellquote (command parsing), yaml.v3 (config)
